Create Users and Admins

User Help
  • Choose User for normal server customers.
  • Choose Admin for full control access.
  • Email can be used as login if your backend supports it.
  • You can later add password reset flow if you want.

Server Resource Defaults

Use these values when creating or editing customer servers.

What this is for
  • Extra ports let a server use more than one port if needed.
  • Database name is useful if you later auto-create MySQL databases.
  • Memory is helpful for Docker container defaults.

Website Integration

Provision users and servers automatically after purchase.

Integration Snippet

<script>
async function provisionOrder(email, serverName, serverType, serverVersion){
  const res = await fetch('REPLACE_PANEL_URL/api/integrations/website/provision',{
    method:'POST',
    headers:{
      'Content-Type':'application/json',
      'x-panel-api-key':'REPLACE_PANEL_API_KEY'
    },
    body:JSON.stringify({email,serverName,serverType,serverVersion})
  });
  const data = await res.json();
  alert(JSON.stringify(data));
  window.location = 'REPLACE_PANEL_URL';
}
</script>
